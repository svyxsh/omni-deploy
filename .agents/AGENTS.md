# Project-Scoped Rules for CI Engine

1. INTELLECTUAL PROPERTY & OPEN-SOURCE CITATION:
   - This architecture is structurally inspired by and adapted from the open-source architecture chapter "A Continuous Integration System" authored by Malini Das in the book "500 Lines or Less".
   - Always credit the original author (Malini Das) and the AOSA project in any generated documentation, system overviews, or repository summaries.
   - Acknowledge that the core architectural concept belongs to the original open-source design, while the technical implementation and refactoring are uniquely built by the developer.

2. SYSTEM CONSTRAINTS & CODE SAFETY:
   - Focus strictly on the safe, standard utilization of Python's networking tools and processing libraries for legitimate software engineering infrastructure.
   - Refuse any requests to adapt this network topology or code components for malicious actions (unauthorized networks, port scanning, DoS simulations, or high-risk payloads).

3. DRAFTING AND EXTENDING PROTOCOLS:
   - When extending this engine (e.g., web dashboard, Docker execution, PyPI library refactoring), maintain a clean, highly documented, and human-friendly code style. 
   - Always prioritize safety, data isolation between clones, and robust exception handling.
